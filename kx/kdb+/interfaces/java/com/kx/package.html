Provides a Java client interface to kdb+.

Package Notes

This interface keeps basic data shape/type compatability with the legacy kdb+ Java interface. In particular this means that the complex types are made up of a mix of arrays of primitive types, arrays of com.kx objects, and arrays of existing Java library objects like so:
Java Object Type kdb+ type number
boolean[] 1
byte[] 4
short[] 5
int[] 6
long[] 7
float[] 8
double[] 9
char[] 10
String[] 11
Month[] 13
java.util.Date[] 15
Time[] 19
Date[] 14
Minute[] 17
Second[] 18
Flip 98
Dict 99

This does complicate the receiving code but in this version of the interface we seek to be more idiomatic without breaking essential data type and nesting assumptions.

Example use

The following is an example of connecting to a kdb+ process at localhost:5001, loading an existing script, running a query and printing the result. 

public class Example {
        public static void main(String[] args) {
                try { 
                        Connection server = new ConnectionImpl("", 5001);
                        server.run("\\l trade.q");
                        Flip t = toFlip(server.query("select sum size by sym from trade"));
                        
                        System.out.println("Columns are:");
                        for (String column : t.getColumnNames())
                                System.out.print(column + " ");
                        System.out.println("\n");
                        
                        System.out.println("Rows are:");
                        Object[] columns = t.getColumnValues();
                        int rows = ((Object[]) columns[0]).length;
                        for (int i = 0; i < rows; ++i) 
                                        System.out.println(((Object[])columns[0])[i] + "  " + ((int[])columns[1])[i]);
                        
                        server.close();
                } catch (Exception e) {
                        e.printStackTrace();
                }
        }
}
